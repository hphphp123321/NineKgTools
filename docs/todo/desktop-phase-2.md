## 1. 识别诊断面板（IdentificationDiagnosticsView）

对应 Web 的 [`Components/Tasks/IdentificationDiagnosticsView.razor`](../../NineKgTools.Web/Components/Tasks/) 和 [`wwwroot/css/components/diagnostics.css`](../../NineKgTools.Web/wwwroot/css/components/) (`diag-*` 类前缀)。

### 任务

- [ ] **VM**：`ViewModels/Components/IdentificationDiagnosticsViewModel.cs`
- [ ] **数据源**：`TaskExecutionInfo.IdentificationDiagnosticsJson` 反序列化为 `IdentificationDiagnostics` 对象
- [ ] **顶栏摘要**：识别耗时 / 总尝试网站数 / 选中网站
- [ ] **时间线**：按时间顺序的 `BeginAttempt → EndAttempt` 段；每段显示状态（成功 / 跳过 / 失败 / 缓存命中）+ Reason
- [ ] **候选列表**：每个网站的 Top 5 候选（来自 `RecordCandidates`），高亮 `MarkChosen` 选中那一项
- [ ] **关键词**：`RecordKeywords` 的搜索词列表
- [ ] **Avalonia 控件**：用 `Expander` 展开每个网站尝试段；候选用 `ListBox` + 自定义 ItemTemplate
- [ ] **错误状态**：诊断数据为 null 时显示 "本次任务未在 SingleSourceIdentificationTask 上下文中执行"
- [ ] **集成点**：嵌入 [Phase 1 任务 4](desktop-phase-1.md#4-后台任务backgroundtaskspage) 的 `TaskDetailsDialog` Tab + [Phase 1 任务 5](desktop-phase-1.md#5-待处理媒体源双-tab) 的待处理 Tab 详情

### 设计稿

```
顶栏摘要（高 64px，pseudo-Card 卡片）
┌────────────────────────────────────────────────────────────┐
│  ⏱ 5.4s 总耗时    🌐 3 个网站    ✓ DLsite 命中 RJ01081508  │
│  关键词: 视频名_部分_关键字 · 视频名_备选                  │
└────────────────────────────────────────────────────────────┘

时间线（Vertical Stack）
┌─ Step 1 ────────────────────────────────────────────────────┐
│ 🟦  DLsite                                          1.2s    │  ← Expander header
│     ✓ 命中 · RJ01081508 · "视频名 完整版"  · 评分 0.95     │
├─────────────────────────────────────────────────────────────┤
│   候选列表（展开后）                                         │
│   ★ RJ01081508  视频名 完整版                       0.95   │  ← MarkChosen 高亮（accent 色）
│     RJ01099876  视频名 简短版                       0.74   │
│     RJ01112233  视频名 衍生作                       0.61   │
│     ...                                                     │
└─────────────────────────────────────────────────────────────┘
┌─ Step 2 ────────────────────────────────────────────────────┐
│ 🟧  Bangumi                                         (跳过)   │  ← 跳过段配灰色
│     ↳ 跳过原因: 优先级低于已命中 DLsite                     │
└─────────────────────────────────────────────────────────────┘
┌─ Step 3 ────────────────────────────────────────────────────┐
│ 🟪  Steam                              💾 缓存命中  0.0s    │  ← 缓存命中独特图标
│     ↳ 缓存键: …                                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键决策

- **状态色映射**：
  - 命中 = `SystemFillColorSuccess` (绿) + ✓ 图标
  - 跳过 = `TextFillColorTertiary` (灰) + ↳ 图标
  - 失败 = `SystemFillColorCritical` (红) + ✕ 图标
  - 缓存命中 = `SystemFillColorAttention` (蓝) + 💾 图标 + "0.0s" 强调
- **`Expander` 默认全展开**——对话框打开时直接看完整时间线；用户可手动折叠
- **候选 Top 5** 表格风：score 列右对齐，3 位小数；`MarkChosen` 的那一行整行 accent fill 浅色背景
- **关键词** 顶栏单行展示前 2 个，超过用 "…" + 点击展开 flyout 看全部
- **左侧 timeline 竖线**：用 `Border BorderThickness="0,0,0,2"` 模拟时间线的视觉连贯性，不需要画 Path
- **复用性**：作为 `UserControl` 而非 dialog 内嵌——Phase 1 任务 4 的 `TaskDetailsDialog` 直接 host 这个 UserControl 当一个 Tab

### 验收

1. 任意识别任务详情对话框打开 → 诊断 Tab 显示完整时间线
2. 失败任务的 Reason 字段完整可读（脱敏后的人类可读消息，不是 stacktrace）
3. 缓存命中段在视觉上明显区别于真实尝试段（不同图标 + 不同 brush）
4. 候选列表的 score 列右对齐，且 chosen 那行高亮明显

### 任务

- [ ] **VM**：`ViewModels/Pages/WebsitesViewModel.cs`，依赖 `Config` + `WebsiteService`
- [ ] **网站卡片**：DLsite / Bangumi / Steam 各一张卡（顶部一行）
- [ ] **优先级拖拽列表**：5 个分类（音频 / 视频 / 游戏 / 图片 / 文本）每类一份独立优先级列表
- [ ] **设置对话框**：每个网站独立 dialog（DLsite 无认证 / Bangumi `ApiKey` / Steam `CountryCode` 下拉但 `cn` 禁用）
- [ ] **保存策略**：变化即时保存（不需要"保存"按钮），右上角浮现"已保存"snackbar；改了重启才生效的字段（如果有）会用 InfoBar 提示
- [ ] **测试连接**按钮：每个网站调一次 `IWebsite.GetMediaInfoAsync(<sample id>)` 验证接通；测试期间按钮 disabled + spinner

### 设计稿

```
顶部网站卡片行（高 140px）
┌──────────────────┬──────────────────┬──────────────────┐
│  🌸 DLsite       │  🔵 Bangumi      │  🟦 Steam        │
│                  │                  │                  │

│                  │                  │                  │
│  [设置] [测试]   │  [设置] [测试]   │  [设置] [测试]   │
└──────────────────┴──────────────────┴──────────────────┘

优先级配置（5 个 Tab，每个 Tab 一份独立列表）
┌──[音频][视频[选中]][游戏][图片][文本]──────────────────┐
│                                                        │
│   视频识别优先级（拖拽排序）                            │
│                                                        │
│   ┌──────────────────────────────────────┐           │
│   │ ⋮⋮  1. DLsite          ●启用  [删除] │  ← 拖拽手柄 + 顺序号
│   ├──────────────────────────────────────┤           │
│   │ ⋮⋮  2. Bangumi         ●启用  [删除] │           │
│   ├──────────────────────────────────────┤           │
│   │ ⋮⋮  3. Steam           ○禁用  [删除] │           │
│   └──────────────────────────────────────┘           │
│                                                        │
│   [+ 加入站点 ▾]                                       │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 关键决策

- **拖拽实现**：用 Avalonia 11 的 `DragDrop` API；拖拽时被拖项 opacity 0.5，目标位置上方显示一条 2px accent 高度的"插入指示线"
- **每分类独立列表**：5 个 Tab 而不是把 5 类塞一个表格——分类间逻辑独立，5 列表格反而难看
- **"删除"按钮** 把网站从该分类的优先级列表移除（不是删整个网站配置）；点击 "+ 加入站点" 把没在列表里的站点加回来
- **设置对话框** 复用 `NineKgConfirmDialog` 骨架但 intent=`Info`，body 是表单：
  - DLsite：空（无认证）
  - Bangumi：单个 `ApiKey` 输入框 + "去 Bangumi 申请 ApiKey" 链接
  - Steam：`CountryCode` ComboBox（cn 项 disabled + tooltip 解释 "Steam 部分游戏对 CN 区屏蔽，会导致 success=false"）
- **测试连接** 成功 → InfoBar Severity=Success 显示 1.5s 后自动消失；失败 → InfoBar Severity=Error 持久化直到用户手动关闭

### 验收

1. 拖拽优先级 → 重启应用 → 顺序保留
2. 修改 Bangumi ApiKey 立刻生效，跑识别新文件 → 日志显示 Bangumi 优先识别
3. Steam CountryCode 下拉里 `cn` 不可选（disabled + tooltip 解释）
4. 测试连接成功显示 InfoBar 自动消失；失败 InfoBar 持久化

```
「外观」分组示意（典型）：
- 主题: ◉ 跟随系统  ○ 浅色  ○ 深色
- 关闭主窗时: ◉ 最小化到托盘  ○ 退出应用
- 媒体库默认视图: ◉ 网格  ○ 列表

「媒体源」分组示意（list 编辑）：

```

### 关键决策

- **双列布局** 优于 `TabControl` —— 配置项 10+ 组，标签页太多变成"标签滚动条"；左侧导航更扫描友好
- **每组顶部一行 H3**（`TextStyle="Subtitle"`） + 下方表单，组与组之间 32px 垂直间距；同组字段间 16px
- **即时保存** 替代"保存"按钮——避免用户改完忘记保存，对应 Win11 设置 App 的范式
- **危险操作分隔** 在左侧导航底部独立区，与正常配置组分开——重置默认 / 清空缓存这种放上面容易误点
- **数据库分组只读** + 显式 warn——避免用户改 db 路径导致数据丢失；导出/导入按钮提供合规迁移途径
- **InfoBar 提示重启字段**：每个组顶部一条 InfoBar，列出本组哪些字段需重启（hardcode 一份字段→需要重启的映射表）
- **新增"清空缓存"和"导出/导入配置"** 是桌面端特有需求——Web 端跑在容器里，缓存清理通过重启容器；桌面端用户期望应用内操作

### 验收

1. 改 `MaxConcurrentIdentificationTasks` 5→10 + 重启 + 跑批量识别 → 实际并发提升到 10
2. 改主题（Light / Dark / Default）实时切换无需重启
3. 删除某 watch_folder + 重启 → MonitorService 不再监控
4. 重置默认走 Destructive 确认 + 之后所有字段恢复到 example 值
5. 清空缓存后 `dataDir/.cache/` 目录消失重建

### 关键决策

- **创作者头像** 没设置时显示 `AcrylicBrush` 背景 + 居中首字母（`Avalonia` 的 `Border` + `TextBlock`），不强制要求每个创作者都有头像
- **AI 推断别名** 节流：单次最多并发 3 个 OpenAI 请求（`SemaphoreSlim(3)`），UI 显示进度 "已处理 3/12 创作者"
- **合并** 只允许把 A 合并到 B，不允许多对多——简化心智模型
- **社团页与创作者页同款 layout**——只是数据来源不同，复用 ViewTemplate
