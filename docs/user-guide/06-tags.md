# 06. 标签系统（`/tags` & `/tag/{id}` & `/tags/mappings`）

## 这个页面是干啥的？

标签系统是**对媒体内容做语义维度归类**——AI 识别时自动打标，你也可以手动管理。三个相关页面：

| 路由 | 干啥 |
|---|---|
| `/tags` | 标签库浏览（所有 TopTag + 子标签） |
| `/tag/{tagId}` | 单标签详情 + 编辑 + 关联媒体 |
| `/tags/mappings` | 识别源原始标签 → 你的标签 的映射规则 |

![标签映射](../assets/screenshots/tag-mappings.png)

## 标签的数据模型

```
TopTag（一级分类）
  ├─ "偏好/需求"
  │    ├─ Tag "ASMR"
  │    ├─ Tag "治愈"
  │    └─ ...
  ├─ "物品/道具"
  ├─ "角色"
  └─ ...8 个一级分类
```

默认 8 个 TopTag + 375 个 Tag——**来源是从老版本 DLsite 网站爬取整理的字典**（含 R18 标签），仅作种子数据，可任意删改。详见 [`docs/tags-yaml-reference.md`](../reference/tags-yaml.md)。

## `/tags` —— 标签库浏览

### 主要操作

#### 我想浏览所有标签

进入 `/tags` → 8 个一级分类卡片左右排列。点其中一个 → 展开该一级分类下的所有子标签 → 每个子标签是 chip 形态，标签名旁有"使用此标签的媒体数"。

#### 我想新建一个标签

页面右上"添加标签"按钮 → 弹对话框：

- 名称（必填，DB 唯一键）
- 描述（可选，影响向量化质量）
- 一级分类（必选，从现有 TopTag 选）
- 保存

#### 我想新建一个一级分类（TopTag）

v1.0 **没有 UI** 加 TopTag——只能改 `Config/tags.yaml` 加节点，**清库**（`NINEKG_RESET_DB=true`）一次让 yaml 重新灌入。详见 [`tags-yaml-reference.md`](../reference/tags-yaml.md)。

v1.1 计划提供 UI 加 TopTag 的入口。

#### 我想跳转到标签映射管理

页面顶部 **"标签映射"** 按钮 → 进 `/tags/mappings`（详见下文）。

## `/tag/{tagId}` —— 单标签详情

### 主要操作

#### 我想看这个标签下挂了哪些媒体

进 `/tag/123`（具体 id 取决于你的 DB）：

- 顶部：标签名 + 描述（直接编辑）
- 中部统计：媒体数量、创建时间
- 底部：关联媒体列表（沿用 `MediaShownView`，可切网格/列表，可筛选/排序）

#### 我想改标签名 / 描述

直接在页面顶部编辑 → 失焦或回车保存。**不需要弹对话框**——这是项目里少有的"内联编辑"页。

> ⚠️ 改名会影响**所有历史标签向量**——重命名后向量描述变了，建议手动触发 `TagVectorSync` 重算。

#### 我想删除标签

页面右上 ⋮ 菜单 → 删除 → 弹 `NineKgConfirmDialog Destructive`：
- 显示该标签关联的媒体数
- 警告"此操作不可撤销"
- 确认后删除

> 删标签**不删媒体**——只解除关系。

#### 我想把这个标签的所有媒体合并到另一个标签

v1.0 没有"合并按钮"——手动两步走：

1. 选中该标签下所有媒体 → 多选 → "添加到标签" → 选目标标签
2. 删掉本标签

详见 [13 工作流-流程四 标签清理](13-workflows.md#流程四标签清理--映射)。

## `/tags/mappings` —— 标签映射管理

### 这是做啥用的

识别源（DLsite/Bangumi/Steam）抓回的原始 tag 字符串，通过五种策略匹配到你的标签库：

```
1. 精确匹配     name == 抓到的字符串
2. 包含匹配     name 是抓到字符串的子串，或反之
3. 规范化匹配   去空格 / 大小写 / 全半角后比较
4. 模糊匹配     编辑距离 ≥ 阈值
5. 向量匹配     上面都不命中时，用 embedding 余弦相似度找最近标签
```

**标签映射** = 在自动匹配之前的"硬规则"——直接强制把"巨乳" → "巨乳/爆乳"。

### 主要操作

#### 我想新建一个映射

页面顶部 **+ 添加映射** 按钮 → 弹对话框：

| 字段 | 说明 |
|---|---|
| 源标签名 | DLsite/Bangumi/Steam 返回的原始字符串 |
| 目标标签 | 从你的标签库选 |
| 优先级 | 数字越大越优先（多个映射规则冲突时） |
| 启用 | 默认 ✅ |
| 描述 | 注释自己看 |

保存。**之后所有识别遇到这个源标签都会自动指向目标标签**——不再创建独立标签。

#### 我想看映射的命中情况

页面顶部 5 个统计卡：

- 总映射数
- 启用 / 禁用计数
- 总命中次数
- 未使用映射（从未命中的）

这能帮你**清理无效映射**——长期 0 命中的可能拼写错了或源已经不返回这个字符串。

#### 我想批量管理映射

数据表多选 + 批量启用 / 禁用 / 删除。

#### 我想清理"长期没用"的映射

页面顶部 **"清理未使用"** 按钮 → 弹对话框选条件：

- 从未使用的（命中次数 = 0）
- 90 天内未使用的

确认后批量禁用或删除（你选）。

### 映射只在识别时起作用

映射 = "下次识别遇到 X 字符串时映射到 Y 标签"。

对**已经存在数据库**的媒体没有追溯效果——它们之前关联的"巨乳" 标签依然是"巨乳"，不会自动变成"巨乳/爆乳"。

要清理历史数据 → 走 [13 工作流-流程四](13-workflows.md#流程四标签清理--映射) 的手动合并步骤。

## 进阶用法

<details>
<summary>标签向量化的细节</summary>

`config.yaml` → `ai.vector.tag.enable: true`：每个标签的 `name + description` 会被 OpenAI Embedding API 向量化，存到 `Database/vectors.db`。

向量用途：

1. **语义搜索**：搜"治愈" 找到向量最近的 "温馨" / "治愈系" 等标签
2. **识别匹配 fallback**：DLsite 返回 "癒し"，5 种文本匹配都不命中 → 向量匹配命中你的"治愈" 标签

**描述写得详细**会让识别匹配更准——多写几句具体场景比单一标题词管用。

修改描述 / 改名 **不会**自动重算向量。手动触发：`/tasks/scheduled → TagVectorSync → 立即触发`。或等 6 小时定时跑。

</details>

<details>
<summary>tags.yaml 跟数据库的关系</summary>

`Config/tags.yaml` 是**初始字典**——**仅在数据库为空时启动一次性灌入**。一旦有标签进了 DB，**之后改 yaml 不再生效**——所有改动走 `/tags` 页面 → DB。

要让 yaml 改动生效 → `NINEKG_RESET_DB=true` 启动一次（**会清掉所有数据**！）。

详见 [`tags-yaml-reference.md`](../reference/tags-yaml.md)。

</details>

<details>
<summary>标签匹配阈值调优</summary>

`config.yaml` → `tag_matching` 段：

```yaml
tag_matching:
  enable_fuzzy_matching: true          # 模糊匹配
  similarity_threshold: 0.7             # 模糊匹配阈值
  enable_contains_matching: true       # 包含匹配
  enable_normalized_matching: true     # 规范化匹配
  max_match_results: 5                 # 多匹配时返回几个
  log_match_details: false             # debug 用，会刷屏
```

调试匹配问题：临时设 `log_match_details: true` → 看 `Logs/` 详细记录每个标签是怎么匹的。

</details>

## 跟其他页面的关系

```
/tags ─── 点 TopTag → 展开子标签 chip
   ├─ 点 chip → /tag/{id}（详情）
   │     ├─ 编辑 / 删除 / 看媒体清单
   │     └─ 媒体点击 → /media/{id}
   └─ "标签映射"按钮 → /tags/mappings
                            ├─ 添加 / 编辑 / 批量启用禁用
                            └─ 清理未使用

/media/{id} 详情页 → 标签 chip 点击 → /tag/{id}

识别流程（自动） → 标签匹配（5 策略 + 映射）→ 关联到 /tag/{id}
```

## 常见问题

### Q：识别后看到一堆没见过的标签

DLsite/Bangumi 返回的原标签**没匹配到现有标签也没映射**时会**直接创建新标签**。

清理方式：

1. `/tags` 浏览找到这些孤儿标签
2. 决定"保留"还是"映射"——
   - 保留：放在合适的 TopTag 下，加描述
   - 映射：去 `/tags/mappings` 加映射规则到现有标签
3. 删孤儿标签

### Q：明明加了映射但还是新建了独立标签

可能性：

1. 映射状态是禁用 → 启用
2. 优先级太低被其他映射覆盖 → 加大数值
3. 已经识别过的媒体 → 映射只对**新**识别生效

### Q：标签描述写多详细才有效

短描述够用：一两句话说清楚"什么场景下用这个标签"。

例如：

> ❌ "治愈"：放松向作品
> ✅ "治愈"：通过温暖、正能量内容让人放松减压的作品（治愈系、慢生活、温馨日常）

第二种向量化后能命中"温馨"、"日常"、"放松"等近义词的查询。

### Q：能不能把标签按权重排序

v1.0 没有标签权重——只有"使用次数"作为隐式排序信号。v1.1 计划加显式 weight 字段。

### Q：能批量给媒体加标签吗

`/media/overview` 多选 → 批量加收藏 ✅ / 批量加标签 ❌（v1.1）。

临时方案：进每个媒体详情页编辑 → 加标签。
